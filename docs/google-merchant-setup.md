## One-time setup (owner actions)

### 1 · Google Search Console — verify the domain
1. Go to https://search.google.com/search-console → Add property → **Domain** → `xanvor.com`.
2. Google shows a TXT record (`google-site-verification=…`).
3. Add it in **Cloudflare → DNS → Add record → TXT** for `xanvor.com`, then click Verify.
4. In Search Console → **Sitemaps** → submit `https://xanvor.com/sitemap.xml`.

### 2 · Google Merchant Center — create + claim
1. Go to https://merchants.google.com → create account:
   - Business name: **XANVOR** · Country: **India** · Currency: **INR**
2. **Website** → enter `https://xanvor.com` → claim is automatic if the same Google
   account already verified the domain in Search Console (step 1).
3. Business info: address (Lal Bagh, Moradabad 244001, UP), phone, GSTIN if asked.

### 3 · Add the product feed
1. Merchant Center → **Products → Feeds → Add primary feed**
2. Country: India · Language: English · Destination: Free listings (+ Shopping ads if running ads)
3. Method: **Scheduled fetch**
   - Feed name: `XANVOR retail`
   - File URL: `https://xanvor.com/merchant-feed.xml`
   - Fetch frequency: daily
4. Fetch now → fix any per-item warnings under **Products → Diagnostics**.

### 4 · Merchant Center settings (under "Your business" / "Tools")
- **Shipping**: India · Free shipping · delivery time ~5–10 business days (match reality)
- **Returns**: link `https://xanvor.com/refund-policy.html` (7-day returns)
- **Contact info**: email + phone, must match the site

### 5 · Checklist before products go live
- [x] HTTPS everywhere
- [x] Prices GST-inclusive and identical on PDP, cart, checkout, feed
- [x] Return / shipping / privacy / terms pages linked in footer
- [x] COD available as payment method
- [ ] Replace `xanvor@upi` placeholder before enabling UPI at checkout
- [ ] Real availability — set `availability` to `out_of_stock` in the feed if a piece sells out

## Notes
- Only the 12 Hot-Serve pieces with retail prices are in the feed. B2B/enquiry-only
  pieces can't be listed (Google requires a purchasable price).
- `identifier_exists = no` because handmade pieces have no GTIN/MPN — this is the
  correct, policy-compliant flag for handicrafts.
- Cloudflare's managed robots.txt prepends AI-crawler blocks; Googlebot is unaffected.
